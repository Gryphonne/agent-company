# Review Unit Tests

## Task

Review the provided unit test class against the original source class and the testing policy. Identify coverage gaps, policy violations, correctness issues, and compilation risks.

## Process

1. Read `testing-instructions.md` and `java-instructions.md`
2. Analyze the original source class to understand what SHOULD be tested
3. Review the test class against the checklist in your agent definition
4. Produce the structured review with a clear verdict

## Input

The user will provide the generated test class and the original source file.

## Output

1. The structured review (coverage, violations, correctness, compilation risks)
2. A verdict: PASS, NEEDS REVISION, or FAIL
3. If NEEDS REVISION: the corrected test class with all issues resolved
